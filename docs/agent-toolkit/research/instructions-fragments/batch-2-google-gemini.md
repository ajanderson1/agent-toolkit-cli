## What I checked — Batch 2 Google/Gemini
- `gemini-cli`: Verified the loader constant in `packages/core/src/tools/memoryTool.ts` (`DEFAULT_CONTEXT_FILENAME = 'GEMINI.md'`) and the `docs/cli/gemini-md.md` doc on `main`. Prior holds — `AGENTS.md` is still opt-in via `context.fileName` in `settings.json`; nothing has changed.
- `qwen-code`: Prior ("QWEN.md only, AGENTS.md needs opt-in") is **stale**. Issue [#2006](https://github.com/QwenLM/qwen-code/issues/2006) was closed by [PR #2018](https://github.com/QwenLM/qwen-code/pull/2018) on 2026-03-02, which added `AGENT_CONTEXT_FILENAME = 'AGENTS.md'` to `memoryTool.ts` and made `currentGeminiMdFilename` an array `[QWEN.md, AGENTS.md]` so both load by default. The published `docs/users/configuration/settings.md` hasn't been refreshed yet but the loader source is the ground truth — flipped from `symlink` to `native`.
- `antigravity`: Strong evidence reversal. Earlier writeups (Atamel, aiengineerguide) say "only `~/.gemini/GEMINI.md` is auto-loaded; `AGENTS.md` is not." That was true pre-1.20.3. The official Google AI Developers Forum changelog for Antigravity 1.20.3 (2026-03-05) explicitly adds AGENTS.md auto-loading alongside GEMINI.md, with GEMINI.md keeping higher precedence on conflict. Since AGENTS.md is now read by default, no pointer needed → `native`. Trail-worthy: `GEMINI.md` still wins on conflict, and nested `AGENTS.md` is gated by a "Load nested AGENTS.md files" setting — but the root-level `AGENTS.md` auto-load is on by default.
- `iflow-cli`: Verified via official `platform.iflow.cn` Memory configuration doc and DeepWiki command reference on `main`. Default is `IFLOW.md` at project root and `~/.iflow/IFLOW.md`; `AGENTS.md` requires `contextFileName` in `~/.iflow/settings.json`. Changelog through May 2026 has no entry adding `AGENTS.md` as a hardcoded default. Symlink stands.
