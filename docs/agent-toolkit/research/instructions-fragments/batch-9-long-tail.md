## What I checked — Batch 9 Long-tail
- `mcpjam`: MCPJam main site (mcpjam.com), GitHub repo (MCPJam/inspector), Product Hunt entry. It's an MCP server testing/debugging inspector (Postman-style for MCP + ChatGPT apps SDK), not a coding harness — no root instruction-file concept. Prior subagent fragment matches: by design.
- `mistral-vibe`: Mistral docs (docs.mistral.ai/mistral-vibe/agents-skills) confirm `AGENTS.md` is the default — loads `~/.vibe/AGENTS.md` + nearest project `AGENTS.md` walking up from cwd in trusted dirs. Native. (NB: project subagent fragment focused on a different question — agent TOML files — and is unrelated to this kind.)
- `openhands`: OpenHands V1 SDK Skill guide (docs.openhands.dev/sdk/guides/skill) explicitly states `load_project_skills()` "automatically finds AGENTS.md, CLAUDE.md, GEMINI.md at workspace root." Surprises vs prior: the older microagents-repo doc focuses on `.openhands/microagents/repo.md`, but V1's project-skill loader auto-loads root `AGENTS.md` too. The directory loader is for the skills concept (different kind); root file = native here.
- `replit`: Replit official docs (docs.replit.com/replitai/replit-dot-md) confirm Agent auto-creates and reads `replit.md` at project root only — no AGENTS.md native read, no global path. Sourcetoad blog confirms community uses symlinks (`ln -s AGENT.md .replit.md`) to bridge. Symlink with target `replit.md`.
- `github-copilot`: GitHub docs (docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-custom-instructions) + GitHub Blog changelog (2025-08-28). CLI + coding-agent surfaces read `AGENTS.md` at repo root by default as "primary instructions." User-scope file is `$HOME/.copilot/copilot-instructions.md` (different name). VS Code surface is the exception: requires `chat.useAgentsMdFile` opt-in setting (noted but not the canonical Copilot harness default).
- `pochi`: Pochi rules docs (docs.getpochi.com/rules) directly state AGENTS.md is supported as a fully-equivalent alternative to `README.pochi.md`, both auto-loaded at workspace root and `~/.pochi/`. Native. Diverges from earlier impression that Pochi was skills/agents-folder only.
- `adal`: SylphAI AdaL CLI listing (codingagents.md/agents/adal/) and docs.sylph.ai homepage list "AGENTS.md project context" as a default capability — auto-loads nearest AGENTS.md at task start with no flags. Public repo (SylphAI-Inc/adal-cli) is docs-only so loader code not directly inspectable, but multiple authoritative docs converge on native behavior. No public evidence of a user/global instruction-file path. Promoting from prior `unknown` (which was specifically about subagents, not instructions).
