## What I checked — Batch 5 Pi/agents-std
- `pi`: Read pi-mono README at `packages/coding-agent/README.md` on `main`; confirmed AGENTS.md loaded by default at cwd + parent walk + `~/.pi/agent/AGENTS.md` global, disabled only via `--no-context-files`. Matches prior.
- `amp`: Sourcegraph's `amp` is one of the co-stewards of the AGENTS.md standard (https://ampcode.com/agent.md redirects to https://agents.md/); community guides (vibecoding, layer5) consistently list Amp among the native readers. No surprises vs prior.
- `cline`: Prior said `.clinerules/` directory only (→ gap). **Overturned this session:** PR #5405 closed unmerged but the Cline team shipped equivalent support in PR #7437 (merged 2025-11-13) — CHANGELOG v3.37.0 entry "Add AGENTS.md support" confirms. Project-root AGENTS.md now natively read; `.clinerules/` still works alongside. Global path remains UI-managed.
- `warp`: Warp docs explicitly state Project Rules are stored in `AGENTS.md` at repo root, filename must be **all caps** (case sensitive — worth flagging for any case-insensitive filesystem assumptions). Global rules are UI-only (Warp Drive Personal > Rules); no global file path.
- `deepagents`: Spent extra time on this — initial signals (DeepWiki "MemoryMiddleware loads AGENTS.md to inject context") suggested native. But https://deepagentsdk.dev/docs/guides/agent-memory and the langchain docs confirm middleware **requires explicit `agentId` and source paths**; project-level memory only auto-creates with `requestProjectApproval` callback. No default project-root auto-load → `gap`. Repo's own `AGENTS.md` is a CONTRIBUTING-style doc, not runtime config.
- `firebender`: Docs explicitly state "automatically discovered when you start a conversation. No configuration in `firebender.json` is required"; search order `./` and `./.firebender/` then parents then `~/.firebender/AGENTS.md` fallback. Landed in v0.15.5 (2026-01-28 changelog). Clear native.
