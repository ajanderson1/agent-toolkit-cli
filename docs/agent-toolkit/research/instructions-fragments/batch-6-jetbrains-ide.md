## What I checked — Batch 6 JetBrains/IDE
- `junie`: Read https://junie.jetbrains.com/docs/guidelines-and-memory.html and https://junie.jetbrains.com/docs/junie-ide-plugin.html. Prior assumed `.junie/guidelines.md` but current docs (2026) flip to `.junie/AGENTS.md` preferred, with project-root `AGENTS.md` as documented fallback and `~/.junie/AGENTS.md` as global. Legacy `guidelines.md` still works but is deprecated. Verdict `native` because root `AGENTS.md` is auto-discovered without config.
- `qoder`: WebFetch on https://docs.qoder.com/user-guide/rules confirmed: rules folder is `.qoder/rules/` for structured rules, but root `AGENTS.md` is explicitly auto-recognized ("Agent will automatically recognize and utilize the rules defined in the file"). No documented global path (project-only).
- `cursor`: https://cursor.com/docs/rules now lists `AGENTS.md` in project root as a plain-markdown alternative loaded automatically. `.cursorrules` legacy is being deprecated; `.cursor/rules/*.mdc` is the rich format; `AGENTS.md` is the zero-config option. Global rules live in Cursor Settings UI, not on disk.
- `kilo`: https://kilo.ai/docs/agent-behavior/agents-md confirms auto-load at workspace root, file-name-must-be-uppercase, `AGENT.md` fallback, nested AGENTS.md supported, AGENTS.md is write-protected. No global path (project-only).
- `roo`: https://roocodeinc.github.io/Roo-Code/features/custom-instructions confirms `AGENTS.md` is auto-loaded by default (`roo-cline.useAgentRules: true` default since v3.47.0, Feb 2026). Global is `~/.roo/rules/` directory (not a single AGENTS.md), but project-scope `AGENTS.md` satisfies native verdict. Note repo archived May 2026 per #5966 fetch — feature shipped before archive.
- `continue`: https://docs.continue.dev/customize/deep-dives/rules and https://docs.continue.dev/customize/rules — no `AGENTS.md` auto-load in current docs. Reads markdown lexicographically from `.continue/rules/` (local) and `~/.continue/rules/` (global). Issue #6716 (filed July 2025) is still a feature request, not shipped. Directory loader → `gap` per brief rule 3.
- `windsurf`: https://docs.windsurf.com/windsurf/cascade/agents-md confirms root `AGENTS.md` is "always-on" with no config. Coexists with `.windsurf/rules/` (structured) and `global_rules.md` (global memory file accessed via UI, different filename, different scope). For the project default, AGENTS.md is the canonical root file.
- `trae`: https://docs.trae.ai/ide/rules and traeide.com/news/6 (v1.3.0) confirm `.trae/rules/project_rules.md` + `.trae/rules/user_rules.md` are the loaded files; AGENTS.md request #1911 closed Dec 2025 with no documented shipping evidence in international Trae changelog. Per brief rule 3, fixed-name file in a directory (not root `AGENTS.md`) → `gap`. Note: forum.trae.cn FAQ mentions importing `AGENT.md`/`CLAUDE.md` but that may be CN-only.
- `trae-cn`: forum.trae.cn FAQ + Trae CN changelog (2026-04-15) explicitly call out "Rules 支持嵌套：子仓目录下的 Rule 文件（包括 AGENTS.md）". Trae CN diverges from international Trae here by recognizing root + nested `AGENTS.md` as auto-attached rules. This is a fresh upstream change (April 2026) that overturns any prior assumption of parity between Trae and Trae-CN.
