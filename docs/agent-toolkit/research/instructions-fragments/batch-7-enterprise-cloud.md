## What I checked — Batch 7 Enterprise/cloud
- `bob`: Read `bob.ibm.com/docs/ide/configuration/rules` and `bob.ibm.com/docs/ide/getting-started/tutorials/start-a-project`. IBM Bob is the AI coding agent IBM shipped in 2026; `/init` writes `AGENTS.md` and Bob auto-applies it at runtime. Global scope uses `.bob/rules/` directories (gap shape for instructions kind) — only the project-scope is satisfied natively.
- `codearts-agent`: Searched Huawei Cloud CodeArts intl docs, JetBrains plugin page, Baidu-wiki, Pandaily/Tiger Brokers public-beta coverage; none name a default root instruction-file convention. Public beta released Feb 2026 with English docs still thin. Reporting honestly as `unknown` rather than fabricating.
- `cortex`: Confirmed via `docs.snowflake.com/en/user-guide/cortex-code/cortex-code-snowsight` and Flexera 2026 setup guide that Cortex Code CLI auto-injects project-root `AGENTS.md`. The `~/.snowflake/cortex/` CLI-config tree (settings.json, mcp.json, skills/, agents/, …) has no documented top-level `AGENTS.md` — global scope is gap-shaped.
- `devin`: `docs.devin.ai/onboard-devin/agents-md` confirms Devin auto-reads project-root `AGENTS.md`. No global `AGENTS.md` — Devin's user/team-scope mechanism is the database-backed "Knowledge" system (`docs.devin.ai/onboard-devin/knowledge-onboarding`), which we can't satisfy with a file pointer.
- `droid`: `docs.factory.ai/cli/configuration/agents-md` documents the exact precedence (project → parents → subfolders → `~/.factory/AGENTS.md` personal override) and "no extra flags required". Clean native case at both scopes.
- `rovodev`: Atlassian's own docs (`support.atlassian.com/rovo/*`) talk mainly about skills and `.review-agent.md`; the king-of-kontext deep-dive on Medium documents both project-root `AGENTS.md` auto-load and global `~/.rovodev/AGENTS.md` auto-load. Cross-checked the code-review docs which only confirm `.review-agent.md` for reviews specifically — AGENTS.md auto-load is the broader CLI behaviour.
- `tabnine-cli`: `docs.tabnine.com` Tabnine CLI quickstart calls out `TABNINE.md` at project root as the file that "loads automatically on every session" — explicitly contrasted with skills and *not* called AGENTS.md. The `.tabnine/guidelines/` tree is a directory of rules files (file-vs-directory rule → no global pointer). Verdict is `symlink` for project scope; global is gap.
- `zencoder`: Feb 2026 changelog (surfaced via search; the page itself returns 403 to anonymous WebFetch) reports CLI-level AGENTS.md resolution; this aligns with Zencoder's broader 2026 push toward the `.agents/skills/` standard and away from legacy `.zencoder/` paths. Treating as `native` for project scope; global remains the `.zencoder/rules/*.md` directory (gap shape). Cite-and-re-verify once docs.zencoder.ai is fetchable.
