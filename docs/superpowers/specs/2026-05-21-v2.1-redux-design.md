# v2.1.0 Redux — Universal-Agent Model + Wizard + Grid

**Status:** Approved, ready for implementation
**Date:** 2026-05-21
**Target release:** v2.1.0 (replacing the wrong-model v2.1.0 that was rolled back)

## Goal

Port `vercel-labs/skills`'s 53-agent catalog to Python, fix the install logic so universal-agents (whose `skillsDir === '.agents/skills'`) don't get redundant per-harness symlinks at global scope, and rebuild the CLI wizard + TUI grid on that catalog.

## Why the redux

The original v2.1.0 hardcoded 5 harnesses (`claude, codex, opencode, gemini, pi`) as equally-symlinked. This is wrong: codex/opencode/gemini are *universal* — they read directly from `~/.agents/skills/<slug>/`. The original implementation created useless `~/.codex/skills/<slug>` symlinks that those harnesses don't read. v2.1.0 was force-pushed off main; this spec replaces it.

## What we learned (re-confirming, since this matters)

From `vercel-labs/skills/src/installer.ts:280-323`:

- `isUniversalAgent(t)` iff `agents[t].skillsDir === '.agents/skills'`.
- **Global install + universal agent:** clone canonical only; skip the per-harness symlink (harness reads canonical directly).
- **Global install + non-universal agent:** clone canonical + create `~/.<harness>/skills/<slug>` symlink.
- **Project install + non-universal agent:** clone canonical + create `<project>/.<harness>/skills/<slug>` **only if** `<project>/.<harness>/` already exists. Otherwise skip — don't pollute unrelated projects with empty `.windsurf/`, `.kiro/` etc. dirs.
- **Project install + universal agent:** clone canonical + create `<project>/.agents/skills/<slug>` symlink (all universal agents share that path).

## Component 1: `skill_agents.py` (NEW)

Direct port of `vercel-labs/skills/src/agents.ts`. One frozen dataclass per agent, in a single immutable dict keyed by skills.sh's agent name (NOT a renamed shortlist).

```python
# src/agent_toolkit_cli/skill_agents.py

import os
from dataclasses import dataclass
from pathlib import Path
from typing import Callable


HOME = Path.home()
XDG_CONFIG = Path(os.environ.get("XDG_CONFIG_HOME") or (HOME / ".config"))
CODEX_HOME = Path(os.environ.get("CODEX_HOME") or (HOME / ".codex"))
CLAUDE_HOME = Path(os.environ.get("CLAUDE_CONFIG_DIR") or (HOME / ".claude"))
VIBE_HOME = Path(os.environ.get("VIBE_HOME") or (HOME / ".vibe"))


@dataclass(frozen=True)
class AgentConfig:
    name: str
    display_name: str
    skills_dir: str               # relative path, project-scope
    global_skills_dir: Path       # absolute path, global-scope
    detect_installed: Callable[[], bool]
    show_in_universal_list: bool = True

    @property
    def is_universal(self) -> bool:
        return self.skills_dir == ".agents/skills"


AGENTS: dict[str, AgentConfig] = {
    "aider-desk": AgentConfig(
        name="aider-desk", display_name="AiderDesk",
        skills_dir=".aider-desk/skills",
        global_skills_dir=HOME / ".aider-desk/skills",
        detect_installed=lambda: (HOME / ".aider-desk").exists(),
    ),
    # ... 52 more entries; full list in plan ...
}


def get_universal_agents() -> list[str]:
    return [n for n, c in AGENTS.items()
            if c.is_universal and c.show_in_universal_list]


def get_non_universal_agents() -> list[str]:
    return [n for n, c in AGENTS.items() if not c.is_universal]


def is_universal(name: str) -> bool:
    return AGENTS[name].is_universal


def detect_installed_agents() -> list[str]:
    return [n for n, c in AGENTS.items() if c.detect_installed()]
```

53 real agents + the `universal` synthetic pseudo-agent (`show_in_universal_list=False`, copying skills.sh's pattern). The `detect_installed` is **synchronous** — Python doesn't need the async wrapper TS uses.

## Component 2: `skill_paths.py` (SLIMMED)

Drops `SUPPORTED_HARNESSES` and `_HARNESS_DIR`. Keeps only `canonical_skill_dir()` and `lock_file_path()`. Per-harness paths now come from `AGENTS[name].skills_dir` / `.global_skills_dir`.

The old name `harness_projection_dir()` becomes `agent_projection_dir(agent_name, slug, *, scope, home, project)` — consumes the catalog. Per-scope branching:

```python
def agent_projection_dir(
    agent_name: str, slug: str, *,
    scope: Literal["global", "project"],
    home: Path | None, project: Path | None,
) -> Path:
    cfg = AGENTS[agent_name]
    if scope == "global":
        return cfg.global_skills_dir / slug
    return (project or Path.cwd()) / cfg.skills_dir / slug
```

## Component 3: `skill_install.plan()` + `apply()` — catalog-aware

`plan()`: same signature as before, except `target_harnesses` is now `target_agents` and references AGENTS.

`apply()` adds the skip-symlink logic from `installer.ts:296-323`:

```python
def apply(plan: InstallPlan, *, home, project, env) -> InstallResult:
    # ... clone canonical as before ...
    
    created: list[Path] = []
    for agent_name in plan.add_agents:
        cfg = AGENTS[agent_name]
        
        # Skip rule 1: global + universal — canonical IS the location
        if plan.scope == "global" and cfg.is_universal:
            continue
        
        # Skip rule 2: project + non-universal + agent dir absent
        if plan.scope == "project" and not cfg.is_universal:
            project_dir = (project or Path.cwd())
            agent_root = project_dir / cfg.skills_dir.split("/")[0]
            if not agent_root.exists():
                continue
        
        link = agent_projection_dir(
            agent_name, plan.slug, scope=plan.scope,
            home=home, project=project,
        )
        link.parent.mkdir(parents=True, exist_ok=True)
        # ... conflict checks + symlink create as before ...
        created.append(link)
    
    # ... remove + lock update as before ...
```

`_current_linked_agents()` mirrors the skip rules: for a slug at global scope, a universal agent counts as "linked" iff canonical exists; for non-universal, iff its `global_skills_dir / slug` is a symlink to canonical.

## Component 4: CLI wizard (UPDATED)

`select_agents_to_add()` rendered in two sections:

1. **Universal (.agents/skills) — always included.** Header with the agents from `get_universal_agents()` listed as bullets. Pre-ticked checkboxes; user can uncheck (skills.sh's wording is "always included" because the canonical satisfies them — but skills.sh's flow does let you deselect; we mirror that). Universal agents at global scope still register their selection in the lock, even though no symlink is created.

2. **Additional agents.** `get_non_universal_agents()` as a checkbox multiselect. Pre-tick agents where `detect_installed()` is True (mirrors skills.sh). Includes a search field (questionary's `checkbox` supports it natively — or fall back to filter by typing).

3. **Scope chooser.** Same as before: `global / project` radio.

`select_slug_to_remove` and `select_remove_mode` carry over unchanged from the previous v2.1.0 design.

## Component 5: TUI grid (REBUILT, simpler than v2.1.0-original)

Columns: `slug | claude | pi | state`. Two interactive cells per row (our explicitly-supported non-universal harnesses); no `universal` column at all. Wide-agent management is CLI territory.

Cell glyphs (carry over):
- `✔` green: agent's symlink exists at the expected location.
- `☐`: no symlink.
- `[+]` pending link.
- `[-]` pending unlink.
- `[!]` drift.

State column unchanged: `clean | dirty | missing | copy`.

**Status indicator in title bar**: "Skill · N items" reads from SkillGrid row count (closes #157).

Keybinds unchanged from v2.1.0-original: `space` toggle, `a` column-toggle, `^s` apply, `^r` refresh, `^z` revert, `t` scope-flip, `q` quit (with discard confirm).

## Component 6: cleanup of v2.0.0 carryover

The v2.0.0 `commands/skill.py` is still a single file. Split it into the package as before (`commands/skill/__init__.py` + per-verb files + `wizard.py`).

`commands/skill/__init__.py`'s `add` verb:
- No `--harness` → wizard runs.
- Multiple `--harness <name>` flags → non-interactive; each name validated against `AGENTS` (not against a hardcoded shortlist).
- `-y` alone → blanket install to all detected agents.
