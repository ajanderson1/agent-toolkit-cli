# v2.2 — Library / Install Split + Git-Backed Canonical

**Status:** Draft for review
**Date:** 2026-05-21
**Target release:** v2.2.0
**Prerequisite:** v2.1.0 shipped (universal-agent model + grid + wizard)

## Goal

Separate **library state** (skill exists on disk) from **install state** (skill is visible to agents). The library is a configurable, git-backed store. Agents see skills only via explicit symlinks. `update` and `push` operate on the library as real git working trees.

## Why this matters

v2.1.0 inherited skills.sh's conflation: `skill add` clones the skill INTO `~/.agents/skills/<slug>/` — which IS the universal-agent location — so the act of cloning is the act of installing for all universal agents. Consequences:

- "Universal bundle off at global scope" has no honest meaning. To turn it off we'd have to delete the canonical clone.
- The canonical is not a git working tree (skills.sh strips `.git/` during install). `update` is wipe-and-reclone. `push` is impossible.
- The location is hardcoded. Users can't relocate the library without OS-level symlink hacks.

v2.2 fixes all three by introducing a neutral library and treating every agent-visible path as a symlink layer.

## What we copy from skills.sh (unchanged)

- **Catalog**: 54 agents, same names, same `skillsDir`, same `isUniversal` rule (`skillsDir === ".agents/skills"`).
- **Per-agent skip rule**: project scope + non-universal + agent root dir absent → skip symlink.
- **Lock file = re-install recipe, not state of record**. Filesystem is authoritative for "which agents see this skill."
- **Project scope can clone independently of global**. Two scopes = two clones = can drift.
- **No `agents:` field in lock entries**. Wizard pre-selection uses a separate `last_selected_agents` UI hint.
- **Lock format choices**: no timestamps in the project-scope lock (merge-conflict avoidance); split `upstream_sha` (GitHub tree SHA, "did upstream change?") from `local_sha` (SHA-256 of on-disk files, "did I drift?").

## What we diverge from skills.sh on

1. **Canonical is a neutral library at `$AGENT_TOOLKIT_SKILLS_ROOT`** (default `~/.agent-toolkit/skills/`), NOT `~/.agents/skills/`. Universal agents reach the library via a symlink at `~/.agents/skills/<slug>` → `$ROOT/<slug>`.
2. **Canonical preserves `.git/`**. Library entries are real git working trees. Edits persist; `update` is `git pull` (three-way merge); `push` is `git push`.
3. **Verbs split library from install**. `add`/`remove` manage library presence. `install`/`uninstall` manage agent visibility.
4. **Universal-bundle toggle** exists as an explicit install target (`--agents universal`). Skills.sh locks universals always-on; we don't.
5. **TUI grid** with per-agent columns surfaces install state per scope.
6. **One-shot `skill migrate`** moves v2.1.0 content into the new library shape.

## Library vs install — the mental model

```
┌──────────────────────────────────────────────────────────────┐
│ Library ($AGENT_TOOLKIT_SKILLS_ROOT, default                 │
│         ~/.agent-toolkit/skills/)                            │
│                                                              │
│  ~/.agent-toolkit/skills/journal-skill/   ← git working tree │
│  ~/.agent-toolkit/skills/notes-skill/     ← git working tree │
│  ...                                                         │
└──────────────────────────────────────────────────────────────┘
        ▲                ▲                  ▲
        │ symlink        │ symlink          │ symlink
        │                │                  │
┌───────┴────────┐ ┌─────┴───────┐ ┌────────┴────────────┐
│ ~/.agents/     │ │ ~/.claude/  │ │ <project>/.agents/  │
│ skills/        │ │ skills/     │ │ skills/             │
│ journal-skill  │ │ journal-... │ │ journal-skill       │
│                │ │             │ │                     │

│  see it here)  │ │  Code sees  │ │  universals)        │
│                │ │  it here)   │ │                     │
└────────────────┘ └─────────────┘ └─────────────────────┘
```

Library = storage shelf. Symlinks = which agents can see which skills.

## Verbs

| Verb | Action |
|---|---|
| `skill add <source>` | Clone upstream into the library at `$ROOT/<slug>`. No symlinks. |
| `skill remove <slug>` | Uninstall from all scopes, then wipe `$ROOT/<slug>`. |
| `skill install <slug> [--agents ...] [--scope global\|project]` | Create agent-visibility symlinks. Idempotent. |
| `skill uninstall <slug> [--agents ...] [--scope ...]` | Remove agent-visibility symlinks. Library untouched. |
| `skill update <slug>` | `git pull` (three-way merge) in `$ROOT/<slug>`. Conflicts surfaced to user. |
| `skill push <slug>` | `git push` from `$ROOT/<slug>`. Requires write access to upstream. |
| `skill list` | Show library state + per-scope install state. |
| `skill migrate` | One-shot. Move v2.1.0 content into library; recreate symlinks. |

Notes:
- `--agents` accepts agent names (`claude`, `pi`), `universal` (the bundle), or `all`.
- Default `--scope` is `global`.
- A later interactive wrapper (e.g. `skill quickstart`) can chain `add` + `install` for the common case. Out of v2.2 scope.

## Canonical = git working tree (the big shift)

Today (v2.1.0): `cleanAndCreateDirectory + cp -r --exclude=.git` — canonical is a `.git`-less folder.

v2.2: `git clone <upstream> $ROOT/<slug>` — canonical retains `.git/`. Consequences:

- **Update**: `agent-toolkit-cli skill update <slug>` runs `git pull` (default `--ff` allowing merge). User's local edits three-way merge with upstream. Conflicts surface and stop the verb.
- **Push**: `agent-toolkit-cli skill push <slug>` runs `git add -A && git commit -m "$(cat <commit-msg-prompt>)" && git push`. Fails clearly if no write access.
- **Reset**: `agent-toolkit-cli skill reset <slug>` runs `git fetch && git reset --hard origin/HEAD`. Recovery escape hatch when things get tangled. (Stretch goal — could land later.)
- **Storage cost**: trivial. Skill repos are small.
- **Dirty-state policy**: `update` with uncommitted edits → use git's normal three-way merge; if conflicts arise, leave them in-tree for user to resolve. No auto-stash.

## Lock format

Two lock files (matches skills.sh's split):

**Global lock** at `~/.agent-toolkit/skills-lock.json` (NOT `~/.agents/`):

```json
{
  "version": 4,
  "skills": {
    "journal-skill": {
      "source": "github:ajanderson/journal-skill",
      "source_type": "github",
      "source_url": "https://github.com/ajanderson/journal-skill",
      "ref": "main",
      "skill_path": null,
      "upstream_sha": "abc123...",     // GitHub tree SHA
      "local_sha": "def456...",        // SHA-256 of on-disk skill folder
      "installed_at": "2026-05-21T10:30:00Z",
      "updated_at": "2026-05-21T10:30:00Z",
      "plugin_name": null
    }
  },
  "dismissed": [],
  "last_selected_agents": ["claude", "universal"]
}
```

**Project lock** at `<project>/skills-lock.json`:

```json
{
  "version": 1,
  "skills": {
    "journal-skill": {
      "source": "github:ajanderson/journal-skill",
      "source_type": "github",
      "ref": "main",
      "skill_path": null,
      "local_sha": "def456..."
    }
  }
}
```

Note: no timestamps, no `installed_at`/`updated_at` in the project lock. Project locks are intended to be committed to git; timestamps would generate spurious merge conflicts.

**No `agents:` field, no `universals_enabled`, no `extra_agents`.** Install state is read from the filesystem (presence of symlinks at known agent paths).

## Filesystem model

### Global scope

- `$ROOT/<slug>/` — canonical git working tree.
- `~/.agents/skills/<slug>` → `$ROOT/<slug>` (created iff universal bundle is installed at global scope).
- `~/<agent-home>/skills/<slug>` → `$ROOT/<slug>` (created iff that agent is installed at global scope).

### Project scope

- `<project>/.agents/skills/<slug>/` — **canonical for this project**. Real git clone, can drift from global. (Same as skills.sh's project-scope canonical.)
- `<project>/<agent-dir>/skills/<slug>` → `<project>/.agents/skills/<slug>` (created per per-agent skip rules).

The library `$ROOT` is global-only. Project scope keeps skills.sh's "each project gets its own canonical" model.

### Install-state detection

`detect_installed_agents(slug, scope)` returns the set of agents currently visible at that scope:

- For each agent in the catalog, compute its target path at this scope.
- `lstat` the path; if it's a symlink resolving to the right canonical, it counts as installed.
- Otherwise, not installed.

This is the SSOT for `list`, the TUI grid, and the wizard's pre-selection defaults.

## Engine API changes

Current `skill_install.plan()` takes `(slug, scope, add_agents, remove_agents)` and returns a delta. v2.2:

```python
@dataclass(frozen=True)
class LibraryAddPlan:
    slug: str
    source: SkillSource
    target_dir: Path           # $ROOT/<slug>
    will_clone: bool           # False if already in library

@dataclass(frozen=True)
class LibraryRemovePlan:
    slug: str
    target_dir: Path
    installed_at_scopes: list[Scope]   # uninstall implied before remove

@dataclass(frozen=True)
class InstallPlan:
    slug: str
    scope: Scope
    add_symlinks: list[SymlinkOp]      # paths to create
    remove_symlinks: list[SymlinkOp]   # paths to remove
    skip_reasons: dict[str, str]       # agent → reason (e.g. "agent root dir absent")

def plan_add(source: str) -> LibraryAddPlan: ...
def plan_remove(slug: str) -> LibraryRemovePlan: ...
def plan_install(slug: str, scope: Scope, agents: list[str]) -> InstallPlan: ...
def plan_uninstall(slug: str, scope: Scope, agents: list[str]) -> InstallPlan: ...

def apply_add(plan: LibraryAddPlan) -> None: ...
def apply_remove(plan: LibraryRemovePlan) -> None: ...
def apply_install(plan: InstallPlan) -> None: ...
def apply_uninstall(plan: InstallPlan) -> None: ...
```

`update` and `push` are pure git operations on `$ROOT/<slug>/`; they don't go through plan/apply.

## CLI surface

```
agent-toolkit-cli skill add <source>
agent-toolkit-cli skill remove <slug>
agent-toolkit-cli skill install <slug> [--agents AGENTS] [--scope SCOPE]
agent-toolkit-cli skill uninstall <slug> [--agents AGENTS] [--scope SCOPE]
agent-toolkit-cli skill update <slug>
agent-toolkit-cli skill push <slug> [-m MESSAGE]
agent-toolkit-cli skill list [--scope SCOPE]
agent-toolkit-cli skill migrate
```

`--agents` is comma-separated. Special values: `universal` (the bundle), `all` (every detected-installed agent at scope). Default: error out and prompt user to specify (no implicit "install everywhere").

## TUI grid (carry forward from v2.1)

Columns: `slug | universal | Claude Code | Pi | (collapsed: +39 more) | source | state`.

- Each agent column shows install state at the current scope as a single character (`●` installed / `○` not / `—` not applicable per skip rule).
- `universal` column toggles the bundle (`~/.agents/skills/<slug>` symlink at scope).
- Cell click = `plan_install` / `plan_uninstall` for that single agent.
- Collapsed "+39 more" expands to a horizontally-scrolling block of individual agent columns.

## Migration (one-shot, v2.1.0 → v2.2)

`agent-toolkit-cli skill migrate` does:

1. **Detect v2.1.0 state**: enumerate `~/.agents/skills/<slug>/` directories that are NOT symlinks.
2. **For each detected slug**:
   - `mv ~/.agents/skills/<slug> $ROOT/<slug>`.
   - `git -C $ROOT/<slug> rev-parse` to check it's a git repo. If not (skills.sh strips `.git/`), re-clone fresh from lock-recorded `source` and replay any uncommitted edits as a patch. (Edge case — flag for review during impl.)
   - `ln -s $ROOT/<slug> ~/.agents/skills/<slug>`.
   - Verify all existing per-agent symlinks (`~/.claude/skills/<slug>` etc.) still resolve through the new symlink chain; they should, but `realpath` each as a sanity check.
3. **Update lock file**: rewrite from `~/.agents/.skill-lock.json` (skills.sh path) to `~/.agent-toolkit/skills-lock.json` (our path); bump version to 4.
4. **Report**: print a summary of moved skills, any per-skill recovery steps needed (e.g. "journal-skill had no `.git/`; re-cloned from `github:.../journal-skill`").

Migrate is interactive — confirms before each `mv`. Provides `--dry-run` and `--yes`.

## Implementation plan

Phased, each phase a separate PR:

**Phase 1: Library primitives + env var.** Introduce `$AGENT_TOOLKIT_SKILLS_ROOT` resolution; refactor `skill_paths` to expose `library_root()` / `library_skill_path(slug)`. No behavior change yet (`add`/`remove` still install for universals). Tests pass.

**Phase 2: Split `add`/`remove` from `install`/`uninstall`.** New verbs. Old behavior preserved behind a compat shim — `add` continues to do "add to library + install for universal bundle at global scope" if `--no-install` not passed. Wizard unchanged. New tests for library-only paths.

**Phase 3: Git-backed canonical.** `add` switches from "clone to tmp + cp" to "git clone into library." `update` becomes `git pull`. `push` lands. Tests for clean/dirty/merge-conflict cases.

**Phase 4: Strict separation.** Remove the compat shim from phase 2 — `add` is library-only, period. Wizard updated to call `add` then `install` separately. Docs updated.

**Phase 5: Migration.** `skill migrate` command. Manual once-through with AJ on his machine; iterate based on what breaks.

**Phase 6: TUI updates.** Grid surfaces per-scope install state. Universal column = bundle toggle. Cell clicks call `plan_install`/`plan_uninstall`.

Each phase has its own design notes if it diverges from this spec.

## Risks

- **Migration is the riskiest piece.** v2.1.0 users have content at `~/.agents/skills/<slug>/` with no `.git/`. Re-cloning to recover loses any local edits. Mitigation: dry-run, interactive confirmation, lock-recorded `source` is authoritative.
- **Git-backed canonical = new failure modes.** Merge conflicts, detached HEAD, dirty tree blocking `update`. Mitigation: `skill reset` escape hatch, clear error messages, document the recovery flow.
- **Two locks (global + project) doubles read/write code paths.** Same as skills.sh; not a new risk.
- **Library env var changes change canonical location.** If a user moves `$ROOT` between invocations, the symlinks break. Mitigation: store the resolved root in the global lock at write time; warn on read if env var disagrees.
